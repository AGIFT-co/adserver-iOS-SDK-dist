# PRIVACY_DISCLOSURE — App Store プライバシー申告ガイド(栄養ラベル記入用)

> **対象読者**: AdServerSDK を組み込むアプリベンダー(App Store Connect の「App のプライバシー」申告を記入する担当者)。
> **本資料の範囲**: **SDK 起因のデータ収集のみ**。アプリ自身が収集するその他のデータは、別途アプリの実態に応じて申告してください。

---

## 1. 前提: PPID は「トラッキング」に該当しません(2026-06-22 方針変更)

AdServerSDK の PPID(会員 ID)は、当社の広告配信サーバーでの配信最適化に利用しますが、**Apple の定義する「トラッキング」(複数事業者のアプリ／サイトを横断したユーザー単位の広告ターゲティング、またはデータブローカーへの提供)には該当しない**との判断です(2026-06-22 更新)。これに伴い:

1. App Store Connect の「**Data Used to Track You(あなたを追跡するために使用されるデータ)**」は **「いいえ」** で申告します。
2. **ATT プロンプトの取得は不要**です。SDK からも ATT 関連 API・`NSUserTrackingUsageDescription` 要件は撤去しました。

> **重要(免責)**: 本資料は当社の上記プライバシー判断に基づく記入指針です。最終的な App Store Connect の申告は、各アプリの実態(PPID の用途・他データとの結合・共有先など)を踏まえたうえで、**アプリベンダー(必要に応じて法務)の責任で確定**してください。アプリ側で PPID を別目的(例: 自社のクロスアプリ解析)に二次利用する場合は、その用途に応じた申告・ATT 対応が別途必要になり得ます。

---

## 2. 申告内容

App Store Connect →「App のプライバシー」での記入指針:

### 2-1. 広告データ(Advertising Data) — **PPID の利用有無に関わらず申告が必要**

SDK は広告の表示(view)・タップ(click)イベント(eventType・adUnitId・creativeId・発生時刻・イベント ID)を計測 API へ送信し、サーバーで保存します。これは Apple の定義する「**広告データ(Advertising Data)**」に該当します(同意・PPID の設定と無関係に常時収集されます)。

| 設問 | 記入内容 |
|------|---------|
| 収集するデータタイプ | **広告データ(Advertising Data)** — 広告の表示・タップイベント |
| データの用途 | **第三者広告(Third-Party Advertising)** — 当社広告の配信実績の計測・請求根拠のため |
| ユーザーへの紐付け | **はい(Linked to the User)** — PPID を併用する場合(イベントに PPID が同載され得るため)。PPID を利用しない構成では特定のユーザーに紐付きませんが、申告の確定はアプリベンダーの責任で行ってください |
| トラッキングへの使用 | **いいえ(Not Used to Track You)** — クロスアプリ追跡に利用しないため(§1 参照) |

### 2-2. ユーザー ID(User ID) — PPID を利用する場合のみ

| 設問 | 記入内容 |
|------|---------|
| 収集するデータタイプ | **ユーザー ID(User ID)** |
| データの用途 | **第三者広告(Third-Party Advertising)** — アプリ内に当社（アプリから見て第三者）の広告を表示し、その出し分けの最適化に用いるため。クロスアプリ追跡・データブローカーへの提供には用いない（トラッキング申告は引き続き「いいえ」・§1 参照） |
| ユーザーへの紐付け | **はい(Linked to the User)** — 会員 ID そのものがユーザー識別子のため |
| トラッキングへの使用 | **いいえ(Not Used to Track You)** — クロスアプリ追跡に利用しないため(§1 参照) |

### SDK が収集**しない**もの(申告不要)

| データ | 説明 |
|---|---|
| デバイス ID(IDFA) | SDK は IDFA を取得・送信しません |
| 位置情報・連絡先・閲覧履歴等 | 取得しません |

> 広告画像の取得・計測ログ送信時に IP アドレス等の通信上の情報がサーバーに到達しますが、これは機能提供に必要な範囲の通信です。

> **補足(送信項目の網羅開示)**: SDK は広告取得リクエスト(/delivery)で `sdkVersion`(SDK の版数)と `appId`(アプリのバンドル ID)を送信します。いずれも個人データではありませんが、送信項目の網羅性のため開示します。

---

## 3. PPID を利用しない場合

アプリが `setPPID()` / `setConsent()` を一切呼ばない(非ターゲティング配信のみ)場合でも、**広告接触データ(view / click イベント)は SDK により収集されます**(§2-1)。この構成ではイベントにユーザー ID が含まれないため、**ユーザーに紐付かない広告データ**としての収集になります。SDK 起因のユーザー ID(PPID)収集は発生しません。

| 構成 | 収集するデータタイプ(SDK 起因) | Data Used to Track You | ATT プロンプト |
|------|------------------------|------------------------|---------------|
| PPID 利用 | 広告データ(view/click) + User ID(同意 ON 時のみ送信) | いいえ | 不要 |
| PPID 不使用 | 広告データ(view/click・ユーザーに紐付かない) | いいえ | 不要 |

## 4. 関連ドキュメント・補足

- 同意の実装手順: [VENDOR_GUIDE.md](VENDOR_GUIDE.md)
- Privacy Manifest(`PrivacyInfo.xcprivacy`)は **SDK に同梱済み**です。トラッキング非該当の判断に伴い、Manifest の `NSPrivacyTracking` は **false**、トラッキングドメインの申告はありません。なお Privacy Manifest を同梱していても、**App Store Connect の申告自体は引き続きアプリベンダーの責務**です。
- 本資料は SDK の実装(送信ゲート・送信項目)と整合するよう保守されます。SDK 更新時は最新版を参照してください。
